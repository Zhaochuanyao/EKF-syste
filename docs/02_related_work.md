# 第二章 相关工作综述

## 2.1 目标检测方法

### 2.1.1 两阶段检测器：Faster R-CNN

Faster R-CNN 是两阶段检测的代表作，由区域提议网络（RPN）和分类回归分支组成。其优点是精度高，缺点是速度慢（GPU 上约 5~10 FPS），难以满足实时跟踪需求，在 CPU 上几乎不可用。

### 2.1.2 单阶段检测器：SSD

SSD（Single Shot Detector）在多尺度特征图上密集预测检测框，省去了 RPN 阶段，速度比 Faster R-CNN 快，但小目标精度较弱。

### 2.1.3 YOLO 系列：本项目选用 YOLOv8n

YOLO（You Only Look Once）系列是当前最流行的实时检测框架。本项目使用 **YOLOv8n**（Ultralytics 发布），其主要特点：

- **速度**：CPU 单帧推理约 50~120 ms（640×640 输入），满足实时演示需求；
- **精度**：在 COCO 数据集 mAP@50 约 37.3%，对行人、车辆等常见类别效果稳定；
- **易用性**：Ultralytics 提供一行调用接口，支持直接加载 `.pt` 权重或导出 ONNX；
- **CPU 友好**：无需 CUDA，安装 `ultralytics` 包即可运行。

与 Faster R-CNN 和 SSD 相比，YOLOv8n 在 CPU 环境下的速度优势显著，是本项目的合理选择。

## 2.2 目标跟踪与状态估计方法

### 2.2.1 线性卡尔曼滤波（KF）

卡尔曼滤波假设系统为线性高斯模型，通过预测步骤和更新步骤交替进行状态估计。线性 KF 通常在目标跟踪中使用匀速直线运动（CV）模型：

- 状态向量：`[cx, cy, vx, vy, w, h]`
- 状态转移：`cx_t = cx_{t-1} + vx * dt`

CV 模型对直线运动目标效果好，但无法建模曲线运动（如车辆转弯），在高机动场景下预测误差大。

### 2.2.2 扩展卡尔曼滤波（EKF）：本项目核心

扩展卡尔曼滤波通过对非线性函数进行一阶泰勒展开（雅可比矩阵线性化）来处理非线性系统。相比于线性 KF，EKF 能够使用更复杂的运动模型（如 CTRV），从而更准确地建模目标的曲线运动。

本项目中的 EKF 实现位于 `src/ekf_mot/filtering/ekf.py`，使用 CTRV 运动模型（详见 [05_ekf_modeling.md](05_ekf_modeling.md)）。

### 2.2.3 SORT（Simple Online Realtime Tracking）

SORT 是一个经典的在线多目标跟踪框架，由 Bewley 等人于 2016 年提出。其核心思路：

1. 用 Kalman Filter（CV 模型）预测各轨迹的下一帧位置；
2. 用 IoU 代价矩阵 + 匈牙利算法关联检测框与轨迹；
3. 按命中/丢失次数管理轨迹生命周期。

SORT 速度快，但在遮挡、目标快速运动场景下 ID 切换频繁，且无法处理外观信息。

### 2.2.4 DeepSORT

DeepSORT 在 SORT 基础上引入 ReID（重识别）外观特征，将外观相似度与运动预测结合进行关联，显著降低了 ID 切换率。但 DeepSORT 需要额外训练 ReID 模型，推理时需要深度网络提特征，对 CPU 资源消耗更高。

## 2.3 本项目的技术选型理由

| 组件 | 选型 | 理由 |
|------|------|------|
| 检测器 | YOLOv8n | CPU 可实时运行，接口简洁，精度满足演示需求 |
| 运动模型 | CTRV | 比 CV 模型更适合车辆等曲线运动目标，EKF 框架扩展性好 |
| 数据关联 | IoU + Mahalanobis + 匈牙利 三阶段 | 融合几何、统计两类代价，比纯 IoU 更鲁棒 |
| 状态估计 | EKF | 适合非线性 CTRV 模型，对比线性 KF 在曲线运动上误差更小 |
| 外观特征 | 未引入（在系统设计中列为扩展项） | 简化系统以聚焦 EKF 主线，CPU 友好 |

### 为什么不用 DeepSORT？

DeepSORT 依赖 ReID 特征提取网络，该网络通常需要 GPU 才能达到实时速度。本项目定位为 CPU 可运行的毕业设计系统，为保证系统可实际运行演示，暂不引入外观特征，后续可作为扩展方向。

### 为什么用 CTRV 而不是 CV？

CV 模型假设目标做匀速直线运动，而视频中的车辆、行人往往存在转弯和变速。CTRV 模型引入了航向角 `theta` 和角速度 `omega`，能够建模曲线轨迹。在 `omega ≈ 0` 时，CTRV 自动退化为 CV 模型，因此 CTRV 是 CV 的严格超集，无额外风险。具体退化逻辑见 `src/ekf_mot/filtering/models/ctrv.py` 第 41~48 行。
