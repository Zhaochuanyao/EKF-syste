# 第一章 研究背景与意义

## 1.1 应用背景

### 1.1.1 智能视频监控

随着城市化进程加快，公共安全对视频监控系统的需求日益旺盛。传统视频监控依赖人工查看录像，效率低下且容易遗漏。智能监控系统通过计算机视觉技术，能够自动完成以下任务：

- **目标检测**：识别画面中的行人、车辆等目标，给出位置（边界框）和类别。
- **多目标跟踪**：在连续帧之间为每个目标分配稳定的唯一 ID，持续追踪其轨迹。
- **轨迹预测**：根据历史运动状态推断目标未来几帧的位置，为碰撞预警、异常行为检测提供依据。

本项目系统能够在普通 CPU 笔记本上实时处理视频流，具备检测、跟踪、预测三项核心能力，可直接应用于轻量级智能监控场景。

### 1.1.2 自动驾驶感知

自动驾驶汽车需要感知周围动态目标（行人、车辆、非机动车），并预测其短期运动趋势，以便规划安全的行驶路径。感知层的核心任务正是"检测 + 跟踪 + 预测"三位一体的处理流水线：

- 检测层提供当前帧各目标的位置和类别；
- 跟踪层跨帧关联，维护每个目标的运动状态（速度、方向、角速度）；
- 预测层根据 EKF 滤波状态递推未来 1/5/10 帧的位置，供决策规划模块使用。

本项目采用的 **CTRV（恒转率恒速度）运动模型** 正是自动驾驶领域常用的车辆运动建模方式，理论来源扎实，工程实用性强。

### 1.1.3 无人机与机器人路径规划

无人机集群协同、地面服务机器人等场景中，系统需要实时感知环境中的移动障碍物并预判其未来位置。运动轨迹预测能够为路径规划算法提供前瞻信息，减少碰撞风险。

---

## 1.2 核心问题

### 1.2.1 为什么需要准确检测目标？

检测是整个系统的输入源头。检测精度直接影响下游跟踪与预测的质量：

- 漏检（False Negative）会导致目标跟踪中断，触发不必要的"轨迹消失"；
- 误检（False Positive）会产生噪声轨迹，干扰真实目标的关联；
- 检测框的位置精度决定了 EKF 观测量的质量，进而影响状态估计收敛速度。

本项目使用 **YOLOv8n**，在 CPU 上单帧推理约 50~120 ms，在速度与精度之间取得了适合毕业设计硬件条件的平衡。

### 1.2.2 为什么需要预测未来轨迹？

纯粹的"当前帧检测"只能告诉你目标现在在哪里，而不能告诉你它将要去哪里。轨迹预测的价值在于：

1. **遮挡恢复**：目标被短暂遮挡时，预测位置可作为"虚拟检测"继续维持轨迹；
2. **ID 一致性**：结合预测位置进行数据关联，能在目标高速运动时减少 ID 切换；
3. **下游应用**：轨迹预测结果可直接供碰撞预警、行为分析模块使用。

## 1.3 本项目的工程意义与应用价值

| 维度 | 说明 |
|------|------|
| 算法完整性 | 从原始视频输入到可视化输出，包含检测、关联、EKF 滤波、轨迹预测完整闭环 |
| 工程规范性 | 模块化 Python 包结构，YAML 配置管理，FastAPI 后端，React 前端，单元测试覆盖核心逻辑 |
| 硬件友好性 | 全流程在 AMD CPU 笔记本上可运行，无需 GPU，适合毕业设计答辩演示 |
| 可扩展性 | EKF 模块、检测器、关联策略均面向接口设计，可替换为更高级的 LSTM/Transformer 预测器或 ReID 模块 |
| 前后端一体 | 提供 Web 界面支持视频上传预测和摄像头实时预测，具备完整的演示价值 |

本项目不仅是一个算法实现，更是一个结构完整、可运行、可演示、可评估的工程系统，覆盖了计算机视觉工程实践的主要环节，具备较高的毕业设计完整度。
